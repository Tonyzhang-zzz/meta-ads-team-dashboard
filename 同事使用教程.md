# Meta Ads MCP 同事使用教程

> 适用对象：需要通过 Codex 查询、分析或监控 Meta Ads 数据的团队成员。
>
> 本教程不要求编程经验。你不需要 Meta Token，不需要登录 Business Manager，也不需要配置 OpenVPN。

## 1. 你将获得什么

配置完成后，可以直接在 Codex 中用中文提问，例如：

- 查询指定广告账户过去 7 天的花费、安装、CPI、购买和 ROAS。
- 找出 CPI 最低或 ROAS 最高的素材。
- 对比最近 7 天和前 7 天，定位指标恶化的广告系列。
- 每天定时检查异常，并生成中文告警。

这是只读服务，不会创建广告、修改预算、暂停广告或更改投放状态。

## 3. 方法一：在桌面版中添加（推荐）

1. 打开 ChatGPT/Codex 桌面应用。
2. 打开左下角或账户菜单中的 **Settings（设置）**。
3. 进入 **MCP servers**。
4. 点击 **Add server**。
5. Name 填写：`meta-ads-team`。
6. Type 选择：`Streamable HTTP`。
7. URL 填写：`https://meta-ads-team-dashboard.onrender.com/mcp`。
8. 保存。
9. 点击 **Restart**，或者彻底退出并重新打开桌面应用。
10. 新建一个 Codex 任务，在输入框中键入 `/mcp`，确认列表里能看到 `meta-ads-team`。

不需要填写 OAuth、用户名、密码或 Meta Token。

## 4. 方法二：使用命令添加

适合已经安装 Codex CLI 的同事。

打开 PowerShell，运行：

```powershell
codex mcp add meta-ads-team --url https://meta-ads-team-dashboard.onrender.com/mcp
```

查看是否添加成功：

```powershell
codex mcp list
```

然后重启 Codex 桌面应用或 IDE 扩展。

如果命令版本不支持 `--url`，可使用下一节的手动配置方法。

## 5. 方法三：手动修改配置文件

Codex 的个人配置文件通常位于：

```text
C:\Users\你的Windows用户名\.codex\config.toml
```

在文件末尾加入：

```toml
[mcp_servers.meta-ads-team]
url = "https://meta-ads-team-dashboard.onrender.com/mcp"
tool_timeout_sec = 120
enabled = true
```

保存文件后重启 Codex。

`tool_timeout_sec = 120` 是为了给 Render 免费实例首次唤醒预留时间。

## 6. 第一次连接测试

在新的 Codex 任务中发送：

```text
使用 meta-ads-team 列出我能访问的 Meta 广告账户。
```

成功时，Codex 会调用 `list_ad_accounts`，返回账户 ID、账户名称、币种、时区和状态。

然后发送：

```text
使用 meta-ads-team 查询 act_2027145677891682 昨天的账户级数据，
返回花费、安装、CPI、购买、购买成本、收入和 ROAS。
```

第一次查询可能等待约 50 秒，这是 Render 免费实例从休眠中启动。若第一次超时，等待 30 秒后原样重试一次。

## 7. 可用工具

| 工具 | 用途 |
|---|---|
| `list_ad_accounts` | 列出共享凭证能访问的所有广告账户 |
| `list_campaigns` | 查看指定账户下的广告系列及状态 |
| `get_ad_insights` | 按账户、广告系列、广告组或广告层级查询成效 |
| `get_creative_performance` | 聚合素材数据，按 CPI、ROAS、花费等排序 |
| `compare_periods` | 对比两个明确日期区间并计算变化率 |

Codex 会根据你的自然语言自动选择工具。通常不需要手动输入工具名，但写出 `使用 meta-ads-team` 可以减少选错数据源的概率。

## 8. 怎样写出清楚的查询指令

一条高质量指令最好包含以下内容：

1. 数据源：`使用 meta-ads-team`。
2. 广告账户：例如 `act_2027145677891682`。
3. 时间：例如昨天、过去 7 天，或明确日期。
4. 层级：账户、广告系列、广告组或素材。
5. 指标：花费、安装、CPI、购买、收入、ROAS 等。
6. 筛选条件：例如安装至少 20、花费至少 100 美元。
7. 输出方式：表格、排名、结论、异常和建议。

通用模板：

```text
使用 meta-ads-team 查询【广告账户】在【时间范围】的【层级】数据。
重点分析【指标】，筛选【条件】，用中文表格输出，
并给出主要结论、异常项和下一步建议。
```

## 9. 常用查询示例

### 查看账户整体表现

```text
使用 meta-ads-team 拉取 act_2027145677891682 过去7天的账户级数据。
用表格展示花费、展示、点击、CTR、安装、CPI、购买、收入和ROAS，
再用3条结论总结整体表现。
```

### 找 CPI 最低的素材

```text
使用 meta-ads-team 查询 act_2027145677891682 过去7天素材表现。
只保留安装数至少20的素材，按CPI从低到高排列前10名，
同时展示花费、安装、购买、收入和ROAS，并标出适合放量的素材。
```

### 找高花费低转化素材

```text
使用 meta-ads-team 查询 act_2027145677891682 昨天的素材数据。
找出花费超过100美元但安装少于10的素材，
按浪费金额排序，并说明建议观察、降量还是暂停。
只提供建议，不要修改广告。
```

### 分析广告系列

```text
使用 meta-ads-team 查询 act_2027145677891682 过去7天的广告系列数据。
按花费从高到低排列，分析CPI和ROAS，
找出消耗占比高但表现低于账户平均值的广告系列。
```

### 对比两个周期

```text
使用 meta-ads-team 对比 act_2027145677891682：
周期一为2026-07-06至2026-07-12，
周期二为2026-07-13至2026-07-19。
比较花费、安装、CPI、购买、收入、ROAS和CTR，
重点解释变化超过20%的指标。
```

### 按国家或平台拆分

```text
使用 meta-ads-team 查询 act_2027145677891682 过去7天数据，
按 country 拆分，比较各国家花费、安装、CPI和ROAS。
过滤掉花费低于50美元的国家。
```

可用拆分字段包括 `country`、`publisher_platform` 等。不要在一次查询中加入过多拆分维度，否则 Meta API 可能拒绝组合。

## 10. 指标口径

- 花费（Spend）：Meta 报告中的广告花费。
- 安装（Installs）：优先使用 Meta 的 `omni_app_install`，没有时使用 `mobile_app_install`。
- CPI：花费 ÷ 安装。
- 购买（Purchases）：优先使用 Meta 聚合购买事件。
- 购买成本：花费 ÷ 购买。
- 收入（Revenue）：Meta 购买事件对应的 action value。
- ROAS：收入 ÷ 花费。
- CTR：点击 ÷ 展示 × 100%。

金额使用广告账户币种。日期以广告账户时区为准，因此“昨天”不一定等于操作者电脑的本地自然日。

## 11. 创建每日监控

先在普通任务中手动运行一次监控指令，确认账户、日期和阈值正确。确认后，再让 Codex 创建定时任务。

推荐监控指令：

```text
创建一个独立的每日监控任务，每天北京时间10:00运行。

每次运行时：
1. 使用 meta-ads-team 拉取 act_2027145677891682 昨天的账户级、广告系列和素材数据。
2. 使用前7天数据作为基准。
3. 如果账户CPI比前7天基准上升20%以上，发出高优先级告警。
4. 如果任一素材花费超过100美元且安装少于10，列入浪费预警。
5. 如果某素材安装至少20且CPI低于账户平均值20%，列入放量候选。
6. 输出中文摘要、异常表格、判断依据和建议动作。
7. 不要修改、暂停或创建任何广告。
8. 如果首次查询因服务唤醒超时，等待5分钟后重试一次。
9. 如果没有明显异常，也要输出“今日无重大异常”和核心指标。
```

创建后，在桌面应用侧边栏的 **Scheduled** 中查看任务状态、下一次运行时间和历史结果。

建议前 3 次运行后检查结果，按业务实际情况调整阈值。例如把 100 美元改为更适合账户体量的金额。

## 13. 常见问题

### MCP 列表里看不到 meta-ads-team

确认保存后点击了 Restart。桌面应用、CLI 和 IDE 扩展共享同一个 Codex 配置，但正在运行的窗口可能不会立即重新加载。

### 提示连接超时

Render 免费实例可能正在唤醒。等待 30 至 60 秒后重试。配置文件中建议设置 `tool_timeout_sec = 120`。

### 提示连接被拒绝或无法访问

确认地址完整包含 `/mcp`：

```text
https://meta-ads-team-dashboard.onrender.com/mcp
```

不要填写 Render Dashboard 管理地址，也不要只填写域名根路径。

### 能列出账户，但目标账户没有数据

先确认账户 ID 是否写成 `act_2027145677891682`。如果账户不在 `list_ad_accounts` 返回结果中，需要服务管理员检查系统用户是否被分配了该广告账户。

### 提示 Meta API 权限或 Token 错误

同事不需要修改本机。把完整错误信息发给服务管理员，由管理员在 Render 和 Meta Business Settings 中检查共享 Token。

### 数据和 Ads Manager 略有差异

检查日期、账户时区、归因设置、数据层级、拆分维度和筛选条件是否一致。Meta 数据也可能存在回传延迟。

### 定时任务没有运行

在 Scheduled 中确认任务处于 Active。桌面端项目型任务可能要求电脑开机并保持应用运行；同时检查该任务是否能使用 `meta-ads-team` 工具。

## 14. 使用边界
